# 2026-07-16 工作日志

## 飞书卡片 Markdown 语法调研（traily MCP 搜索）

使用 traily 连接器（Tavily MCP）搜索飞书卡片 markdown 语法支持情况，解决用户多级标题无法渲染的问题。

### 核心结论
- 飞书卡片有 **Card JSON 1.0**（旧版默认）和 **Card JSON 2.0**（新版）两个版本
- **Card JSON 1.0** 的 markdown 组件仅支持子集：加粗、斜体、删除线、列表、代码块、链接、图片、分割线、@人、emoji
  - **不支持**：标题（#）、引用（>）、行内代码、表格
- **Card JSON 2.0** 支持除 SetextHeading/CodeBlock/HTMLBlock 外所有标准 Markdown 语法（CommonMark）
  - **标题支持 1~6 级**（# ~ ######），字号梯度 26/22/20/18/17/14px
  - 需飞书客户端 ≥ 7.20
- 多级标题无法渲染的根因：使用了 Card JSON 1.0 结构

### 解决方案
在卡片 JSON 顶层添加 `"schema": "2.0"`，并将 body 结构升级为 v2 格式：
```json
{"schema":"2.0","config":{"width_mode":"fill"},"body":{"elements":[{"tag":"markdown","content":"# 标题\n## 二级标题"}]}}
```
